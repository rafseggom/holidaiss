Documentación de API REST de HolidAISS

Endpoint: https://holidaiss.appspot.com/api

Hemos diseñado nuestra API REST enfocada a los servicios LIBRO y LIBRERÍA, según los cuales, una librería está configurada para almacenar distintos libros. Las representaciones JSON de los servicios son las siguientes:

Una representación JSON de LIBRO:

{ "id:" "b3", "title": "Maze Runner", "autor": "James Dashner", "saga": "Maze Runner", "year": "2009" }

Una representación JSON de LIBRERÍA:

{ "id": "l5", "name": "AISSLibrary", "description": "AISS Library", "books": [{ "id:" "s0", "title": "Harry Potter and the Philosopher's Stone", "autor": "J.K.Rowling", "saga": "Harry Potter", "year": "1997" }, { "id:" "s1", "title": "A Game of Thrones", "autor": "George R.R.Martin", "saga": "Game of Thrones", "year": "1996" }, { "id:" "s2", "title": "The Fellowship of the Ring", "autor": "J.R.R.Tolkien", "saga": "The Lord of the Rings", "year": "1954" }] }

Tabla del servicio LIBRO

HTTP

URL

Descripción
GET
/books

Devuelve todos los libros de la aplicación

GET
/books/{id}

Devuelve el libro con el id indicado

POST
/books

Añade un libro cuyos datos se pasan en el cuerpo de la petición en formato JSON. De ser añadido correctamente, devuelve "201 Created". Si no, "error 404"

PUT
/books

Actualiza el libro de datos "title", "autor", "saga" y "year", que se pasan en el cuerpo de la petición en formato JSON, incluyendo el id de dicho libro. Si se realiza correctamente, devuelve "204 No Content". De no ser así, "error 404"
DELETE
/books/{id}

Elimina el libro con id={id}. Si se realiza correctamente, devuelve “204 No Content” Si el libro no existe, devuelve “error 404"

Tabla del servicio LIBRERÍA

HTTP

URL

Descripción
GET
/libs

Devuelve todas las librerías de la aplicación

GET
/libs/{id}

Devuelve la librería con el id indicado

POST
/libs

Añade una librería cuyos datos se pasan en el cuerpo de la petición en formato JSON. De ser añadida correctamente, devuelve "201 Created". Si no, "error 404"

PUT
/libs

Actualiza la librería de datos "name", "description" y "books", que se pasan en el cuerpo de la petición en formato JSON, incluyendo el id de dicha librería. Si se realiza correctamente, devuelve "204 No Content". De no ser así, "error 404"
DELETE
/libs/{id}

Elimina la librería con id={id}. Si se realiza correctamente, devuelve “204 No Content” Si la librería no existe, devuelve “error 404"
